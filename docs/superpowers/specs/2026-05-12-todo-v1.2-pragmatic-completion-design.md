# Spec: TODO v1.2 — Pragmatic Completion (Code-only)

**Date:** 2026-05-12
**Branch strategy:** `feature/aaa-v1.2-pragmatic-completion` → PR to `main` → merge when CI green.

## Goal

Execute every TODO item from `TODO.md` that can be completed with code, tests, configuration, content writing, and documentation. Mark items requiring external accounts, contracts, money, hardware, or other humans as `REQUIRES EXTERNAL ACTION` with a one-line justification.

## Out of scope (will be documented as REQUIRES EXTERNAL ACTION)

- Trademark, EULA, privacy policy, ESRB/PEGI rating (legal)
- Steamworks, GOG, Epic, itch.io, Nintendo/Xbox/PlayStation publishing (commercial)
- Code signing certificates (Apple Developer, Windows EV cert), notarization
- Human voice actors, professional translators (10+ languages), human playtesters
- Sentry account, Zendesk account, Discord moderation
- Marketing, PR, streamer outreach

## Phased execution

### Phase 0 — Critical bugs (must fix first)

1. **Reproduce headless** — `SDL_VIDEODRIVER=dummy` smoke test that renders 1 frame and asserts not uniform.
2. **Debug white screen** — likely cause: rendering order in `core/game.py` (`background.render` → `lighting.render` → entities → HUD). Hypothesis to verify first: `lighting.py` `additive_blit` may be clearing the canvas, or `background.fill` runs after entities.
3. **Fix settings menu non-interactive** — verify event handler returns action codes for Enter/Space/click on Volume and Fullscreen; arrows adjust value.
4. **Audit module integration** — grep for every `systems/*.py` import in `core/game.py` and `main.py`; flag dead modules.
5. **HUD vs gameplay** — verify minimap shows entities, quick inventory updates on pickup.

### Phase 1 — Audit 36 new modules

For each module (`utils/display`, `utils/config`, `systems/feedback`, `systems/tutorial`, `systems/codex`, …, `entities/merchant`):

- Confirm imported from `core/game.py` or `main.py`.
- Confirm at least one runtime call path exists.
- Add a unit test in `tests/test_module_integration.py` exercising the public API.
- Note any dead code in TODO.

### Phase 5 — UX polish

### Phase 6 — Quality

- Smoke visual test (frame hash within tolerance).

### Phase 7 — Build pipeline (only code-doable parts)

### Phase 8 — Docs (code-doable parts)

## Test strategy

- All new test files headless (`SDL_VIDEODRIVER=dummy`, `SDL_AUDIODRIVER=dummy`).
- Existing suite (64 tests) must remain green.
- New test files: `tests/test_phase0_bugs.py`, `tests/test_module_integration.py`, `tests/test_save_fuzz.py`, `tests/test_perf.py`, `tests/test_smoke_render.py`.

## Merge gate

1. All local tests pass on Windows (`python -m unittest discover -s tests`).
2. Push branch.
3. Open PR.
4. CI green on Ubuntu/Windows/macOS × Python 3.10/3.11/3.12.
5. Auto-merge to `main`.

## Risk

- White-screen bug may be in a hard-to-find rendering order. Fallback: add a debug renderer that draws a solid color and confirm it appears.
- 1M-context modules may exceed time budget. Mitigation: prioritize Phase 0 → Phase 1 → quality (Phase 6) → docs (Phase 8) → visual/audio polish last.
- Some Phase 1 modules may have latent bugs; documenting them in TODO is acceptable rather than rewriting blindly.
